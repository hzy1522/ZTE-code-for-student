# GUI Agent 算法设计说明文档

## 1. 概述

本项目实现了一个基于多模态大模型的 GUI Agent，用于完成手机端的自动化操作任务。Agent 通过分析手机屏幕截图，理解用户指令，并生成相应的操作动作（点击、输入、滚动、打开应用、完成）等。

## 2. 系统架构

### 2.1 整体架构

```
用户指令 + 手机截图
       ↓
   AgentInput
       ↓
┌─────────────────┐
│  generate_messages  │
│  - 构建系统提示词   │
│  - 添加历史上下文   │
│  - 编码截图        │
└────────┬────────┘
         ↓
┌─────────────────┐
│   LLM API 调用   │
│  (Volcano Engine) │
└────────┬────────┘
         ↓
┌─────────────────┐
│  _parse_output  │
│  - 解析动作类型   │
│  - 提取参数      │
└────────┬────────┘
         ↓
   AgentOutput
   (action + parameters)
```

### 2.2 核心组件

1. **BaseAgent**: 基础抽象类，定义 Agent 接口和 API 调用保护机制
2. **Agent**: 具体实现类，继承 BaseAgent，实现 `act()` 方法
3. **AgentInput/AgentOutput**: 数据结构，封装输入输出

## 3. 核心算法

### 3.1 消息生成 (generate_messages)

```python
def generate_messages(self, input_data: AgentInput) -> List[Dict[str, Any]]:
    # 1. 构建系统提示词
    system_prompt = _build_system_prompt(instruction)

    # 2. 构建用户消息
    user_content = [
        {"type": "text", "text": f"用户任务: {instruction}"},
        {"type": "text", "text": f"历史动作: {history_summary}"},
        {"type": "image_url", "image_url": {"url": base64_image}}
    ]

    return [
        {"role": "system", "content": system_prompt},
        {"role": "user", "content": user_content}
    ]
```

**关键设计**:
- 系统提示词包含详细的坐标系统说明
- 添加最近3步历史动作摘要，帮助模型理解进度
- 使用 base64 编码图片

### 3.2 输出解析 (_parse_output)

采用多种解析策略，按优先级尝试：

1. **函数调用格式**: `click(point="<point>500 300</point>")`
2. **赛题指定格式**: `CLICK:[[500, 300]]`
3. **Action 章节格式**: `Action: CLICK(point="...")`
4. **中文描述格式**: `点击 <point>x y</point> 位置`

```python
def _parse_output(self, raw_output: str) -> tuple:
    # 方式1: 函数调用格式
    for action_name, pattern in func_patterns.items():
        match = re.search(pattern, raw_output, re.IGNORECASE)
        if match:
            return action_name, _parse_funcall_params(...)

    # 方式2: Action 章节格式
    ...

    # 方式3: 中文描述中提取坐标
    point_match = re.search(r'<point>\s*([\d.]+)\s+([\d.]+)\s*</point>', raw_output)
    if point_match:
        return ACTION_CLICK, {"point": [x, y]}
```

### 3.3 坐标系统

- 屏幕坐标范围: **[0, 1000]** 归一化
- 坐标系: 左上角为原点，x 轴向右，y 轴向下
- **y轴向下增长**：屏幕底部y值接近1000，顶部y值接近0
- 点击目标: 元素中心点的坐标

**归一化坐标参考**:
- y ≈ 50-100: 屏幕顶部（如顶部导航栏、搜索栏）
- y ≈ 400-600: 屏幕中部（如列表内容）
- y ≈ 900+: 屏幕底部（如底部导航栏）

## 5. 关键设计决策

### 5.1 安全性设计

- **配置签名验证**: 防止运行时篡改 API 配置
- **敏感参数过滤**: 禁止通过 kwargs 传入敏感参数
- **受保护的 API 调用**: 所有 API 调用必须通过 `_call_api()` 方法

### 5.2 确定性输出

设置 `temperature=0` 确保每次 API 调用产生相同的结果，便于调试和复现问题。

### 5.3 健壮的解析器

解析器支持多种输出格式，包括：
- 标准函数调用格式
- 赛题指定格式
- 中文描述格式（从叙述文本中提取坐标）

### 5.4 错误处理

```python
try:
    response = self._call_api(messages)
    action, params = self._parse_output(response.choices[0].message.content)
except Exception as e:
    logger.error(f"[Agent] Error calling API: {e}")
    return AgentOutput(action="", parameters={}, raw_output=f"Error: {str(e)}")
```

## 6. 测试结果

### 6.1 性能指标

- **步骤准确率**: ~68%
- **用例准确率**: ~18% (2/11 测试用例通过)

### 6.2 通过的测试用例

- step_aiqiyi_onekey_0011
- step_baidumap_onekey_0008

### 6.3 性能分析

步骤准确率约 68% 与用例准确率约 18% 是一致的：
- 一个用例平均约 10 步
- 全对的概率约为 0.68^10 ≈ 2.1%
- 实际 18% 高于此理论值，说明部分用例步骤较少或较简单，或存在随机性

## 7. 改进方向

1. **更精确的坐标预测**: 引入目标检测模型辅助定位，或使用 few-shot learning 提供示例
2. **多轮对话增强**: 利用历史对话上下文改善理解
3. **自适应策略**: 根据任务类型选择不同的操作策略
4. **错误恢复**: 添加失败重试和回退机制
5. **屏幕元素识别**: 结合 UI 元素检测模型提高元素定位精度
